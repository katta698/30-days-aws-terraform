# Day 1 - Introduction to Terraform and IaC


## What I Learned
- Terraform is a tool by HashiCorp to manage infrastructure as code
- Infrastructure can be defined using configuration files instead of manual setup
- Terraform uses a declarative approach

## Problems with Manual Provisioning
- Hard to track changes
- Prone to human errors
- Not scalable
- Difficult to replicate environments

## Key Benefits of Terraform
- Automation of infrastructure
- Consistency across environments
- Version control using GitHub
- Plan before apply reduces risk

## Terraform Workflow
1. terraform init
2. terraform plan
3. terraform apply

## My Takeaway
Infrastructure should be treated like code, not manual steps.
